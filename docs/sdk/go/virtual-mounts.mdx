Mount a filesystem implemented in Go inside the sandbox: register a provider with `WithVirtualMount`, and every read, write, and directory listing the guest performs under that path becomes a call into your code. See [Virtual Mounts](/sandboxes/virtual-mounts) for concepts and a walkthrough.

The provider API lives in the `github.com/superradcompany/microsandbox/sdk/go/vfs` package.

## Typical flow

```go
package main

import (
    "context"

    m "github.com/superradcompany/microsandbox/sdk/go"
    "github.com/superradcompany/microsandbox/sdk/go/vfs"
)

// A read-only filesystem with one file, backed by a Go map.
type memFS struct {
    vfs.ReadOnly
    files map[string][]byte
}

func (f *memFS) GetAttr(p []byte) (vfs.Attr, error) {
    if string(p) == "/" {
        return vfs.DirAttr(0o755), nil
    }
    if data, ok := f.files[string(p)]; ok {
        return vfs.FileAttr(0o644, uint64(len(data))), nil
    }
    return vfs.Attr{}, vfs.ENOENT
}

func (f *memFS) ReadDir(p []byte) ([]vfs.DirEntry, error) {
    if string(p) != "/" {
        return nil, vfs.ENOTDIR
    }
    return []vfs.DirEntry{{Name: []byte("hello.txt"), Kind: vfs.File}}, nil
}

func (f *memFS) Read(p []byte, offset uint64, size uint32) ([]byte, error) {
    data, ok := f.files[string(p)]
    if !ok {
        return nil, vfs.ENOENT
    }
    if offset >= uint64(len(data)) {
        return nil, nil // empty result signals EOF
    }
    end := min(offset+uint64(size), uint64(len(data)))
    return data[offset:end], nil
}

func main() {
    ctx := context.Background()
    provider := &memFS{files: map[string][]byte{"/hello.txt": []byte("hi from Go\n")}}

    sb, err := m.CreateSandbox(ctx, "demo",
        m.WithImage("alpine"),
        m.WithVirtualMount("/data", provider),
    )
    if err != nil { /* ... */ }
    defer sb.Close() // tears the provider's socket server down too

    out, _ := sb.Exec(ctx, "cat", []string{"/data/hello.txt"}) // served by memFS.Read
    _ = out
}
```

## Options

---

#### <span className="msb-recv"></span><span className="msb-hn">WithVirtualMount()</span>
<div className="msb-tags"><span className="msb-tag is-builder">option</span></div>

```go
func WithVirtualMount(guestPath string, provider vfs.PathFs) SandboxOption
```

Mount a programmable filesystem at `guestPath`, served by `provider`. The SDK hosts the provider on a host Unix-domain socket for the sandbox's lifetime; the runtime connects at boot and serves it as a virtio-fs share.

The guest path must be absolute, must not be `/`, and must not collide with another virtual mount or volume. Because the provider runs in this process, the mount is only valid while the returned Sandbox is open: `Close` tears it down, `Detach` is refused, and combining with `WithDetached` is rejected at create with `ErrInvalidConfig`.

## The PathFs interface

A `vfs.PathFs` answers filesystem operations addressed by **absolute guest path**. Paths are raw `[]byte` (never assumed UTF-8), always begin with `/`, and the root is `/`. Errors must be Linux errno values — return the `vfs.Errno` constants directly; any other error surfaces to the guest as `EIO`. Requests arrive one at a time per connection, so no internal locking is needed for the calls themselves (guard your own shared state as usual).

### Required: a readable tree

```go
GetAttr(path []byte) (Attr, error)               // attributes, or ENOENT
ReadDir(path []byte) ([]DirEntry, error)          // children, excluding "." and ".."
Read(path []byte, offset uint64, size uint32) ([]byte, error)
```

`Read` returns up to `size` bytes from `offset`; a short (or empty) result signals end-of-file.

### Optional: mutations, links, xattrs

```go
Write(path []byte, offset uint64, data []byte) (int, error)
Create(path []byte, attr Attr) (Attr, error)
Mkdir(path []byte, mode uint32) (Attr, error)
Remove(path []byte) error                         // file/symlink/special node
Rmdir(path []byte) error                          // empty directory
Rename(from, to []byte, flags uint32) error       // flags: Linux renameat2 values
SetAttr(path []byte, attr Attr, valid uint32) (Attr, error)
Symlink(path, target []byte) (Attr, error)
ReadLink(path []byte) ([]byte, error)
SetXattr(path, name, value []byte, flags uint32) error
GetXattr(path, name []byte) ([]byte, error)
ListXattr(path []byte) ([][]byte, error)
RemoveXattr(path, name []byte) error
Flush(path []byte) error
Fsync(path []byte, datasync bool) error
FsyncDir(path []byte) error
StatFs() (StatFs, error)
```

Leave any of these on the [`ReadOnly`](#vfs-readonly) defaults and the guest receives `ENOSYS` ("operation not supported") for that operation.

---

#### <span className="msb-recv">vfs.</span><span className="msb-hn">ReadOnly</span>
<div className="msb-tags"><span className="msb-tag is-type">type</span></div>

```go
type ReadOnly struct{}
```

Embeddable base supplying `ENOSYS` implementations of every mutating method, no-op durability (`Flush`/`Fsync`/`FsyncDir`), and a generic `StatFs`. Embed it and implement only `GetAttr`, `ReadDir`, and `Read` for a read-only mount — plus whichever writes you want to support:

```go
type MyFs struct{ vfs.ReadOnly }

func (MyFs) GetAttr(path []byte) (vfs.Attr, error) { /* ... */ }
```

#### <span className="msb-recv">vfs.</span><span className="msb-hn">Attr</span>
<div className="msb-tags"><span className="msb-tag is-type">type</span></div>

```go
type Attr struct {
    Kind  NodeKind
    Mode  uint32     // permission bits, e.g. 0o644 (type bits come from Kind)
    Size  uint64
    Uid   uint32
    Gid   uint32
    Nlink *uint64    // nil: defaulted (2 for dirs, 1 otherwise)

    Atime *time.Time // nil: current time
    Mtime *time.Time // nil: current time
    Ctime *time.Time // nil: current time
}

func FileAttr(mode uint32, size uint64) Attr // regular-file shorthand
func DirAttr(mode uint32) Attr               // directory shorthand
```

---

#### <span className="msb-recv">vfs.</span><span className="msb-hn">Errno</span>
<div className="msb-tags"><span className="msb-tag is-type">type</span></div>

```go
type Errno int32
```

A Linux errno as a Go error. The wire protocol and the guest both speak Linux errno, so providers return these constants directly — no host translation is involved:

`EPERM` `ENOENT` `EIO` `ENXIO` `EBADF` `EAGAIN` `EACCES` `EEXIST` `ENOTDIR` `EISDIR` `EINVAL` `EFBIG` `ERANGE` `ENAMETOOLONG` `ENOSYS` `ENOTEMPTY` `ELOOP` `ENODATA` `EOPNOTSUPP` `ESTALE`

Return `vfs.ENOENT` for a missing path, `vfs.ENODATA` for a missing xattr, `vfs.ENOTEMPTY` from `Rmdir` on a non-empty directory, and so on. A plain (non-`Errno`) error is reported to the guest as `EIO`.
